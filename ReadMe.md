# Clang Database Project

## What This Is

A light-weight C/C++ repository-level symbol database, based on
clang frontend.

## Objective: Using Together With 'cppbind'

`cppbind` is a python binding library in C++ (similar to `pybind11`).
It currently requires the user to provide **return type** and
**parameter list** of a method to expose it in Python.

```c++
// [recap] a simple cppbind example
long add64(long a , long b) { return a + b; }

// written by user.
// The return type and parameter list are explicitly provided,
// but can be automatically synthesized by parsing the file.
gen_modinit_fn_from_fns(
    /* name */ myabc, nullptr, nullptr, nullptr, nullptr,
    gen_builtin_function_def(add64, ":returns: sum of two integers", long, long,
                             long));
```

Using the database generated by this library,
we can automatically create python binding for existing C/C++ libraries.

## struct(s) and class(es)

> In a table named `class`
> schema `class varchar, name varchar, type varchar`

Objective: map a class to its methods and data-members.

1. C++ struct/class: store all prefix namespaces, methods and data-members,
as illustrated below:

```c++
namespace Std {

// After mangling: N3Std3Foo
struct Foo {

  // whole name: _ZN3Std3Foo5valueEv
  // In database: {"class": "N3Std3Foo", "name": "5valueEv", "type": "Ri"}
  // The "type" attribute indicates that it returns `int&`
  int &value(void);

  // whole name: _ZNK3Std3Foo5valueEv
  // In database: {"class": "N3Std3Foo", "name": "K5valueEv", "type": "RKi"}
  // 'K' in both names indicates that the method is const-qualified.
  // The "type" attribute indicates that it returns `const int&`
  const int &value(void) const;

  // whole name: _ZN3Std3Foo3valE
  // In database: {"class": "N3Std3Foo", "name": "3valE", "type": "i"}
  // It is not a method, because the parameter list (after 'E' in its
  // name) is missing. 
  // The "type" attribute indicates that it is an integer data member.
  int val;

  // constructor. whole name: _ZN3Std3FooC1Ei
  // In database: {"class": "N3Std3Foo", "name": "C1Ei", "type": "v"}
  Foo(int);

  // destructor. whole name: _ZN3Std3FooD1Ev
  // In database: {"class": "N3Std3Foo", "name": "D1Ev", "type": "v"}
  ~Foo();

  // whole name: _ZN3Std3Foo5inner
  // In database: {"class": "N3Std3Foo", "name": "5Inner", "type": "N3Std3Foo5Inner"}
  // Set its type to the whole name.
  struct Inner {};

  // whole name: _ZN3Std3Foo5innerE
  // In database: {"class": "N3Std3Foo", "name": "5innerE", "type": "N3Std3Foo5inner"}
  Inner inner;

  // whole name: _ZN3Std3Foo8getInnerEv
  // In database: {"class": "N3Std3Foo", "name": "8getInnerEv", "type": "RN3Std3Foo5Inner"}
  Inner &getInner();
};

} // namespace Std
``` 

2. Mangle C struct name as if it were seen in `namespace extern`.

3. C struct has no member functions. Treat static member functions
as normal functions, stored in table `func`.

## namespace(s)

> In the table named `namespace`.
> schema `name varchar, child varchar, type varchar`

Map outer namespace(s) to its direct inner ones.
The `name` column is the whole name of the outer namespace, 
and the `child` column is the name of the direct inner namespace or class.

Example:

```c++
// {"name": "", "child": "5outer", "type": "N5outer"}
// {"name": "N5outer", "child": "6middle"}
// {"name": "N5outer", "child": "3Foo"}
// {"name": "N5outer", "child": "1fEi", "type": "v"}
namespace outer {

// {"name": "N5outer6middle", "child": "5inner"}
namespace middle {
namespace inner {}
} // namespace middle

// After mangling: N5outer3Foo
struct Foo {
...
};

void f(int);

} // namespace outer

// {"name": "", "child": "1fi", "type": "v"}
void f(int);

// {"name": "", "child": "1x", "type": "i"}
int x;

// {"name": "", "child": "1y", "type": "Ki"}
const int y;
```

## C Symbols

```c
// We handle each extern "C" as if it means 
// `namespace extern`.

extern "C" {
// {"name": "N6extern", "child": "4mainEv", "type": "i"}
int main(void);

struct point2d {
  // whole name: _ZN6extern7point2d1xE
  int x;
  // whole name: _ZN6extern7point2d1yE
  int y;
};
}
```

## Invariants

For the `class` table, the whole name of its member is computed as:

```c++
std::string whole_name(std::string class_name, std::string member_name) {
  return "_Z" + class_name + member_name;
}
```

For the `namespace` table, the whole name of its child is computed as:

```c++
std::string whole_name(std::string namespace_name, std::string child_name) {
  if (namespace_name.empty()) {
    return "_Z" + child_name;
  }
  return "_Z" + namespace_name + child_name;
}
```
